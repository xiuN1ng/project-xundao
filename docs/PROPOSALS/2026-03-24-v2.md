### P1 - 重要问题

#### 8. 【后端】世界BOSS路由名大小写不匹配 ⚠️
- **问题**: `WorldBossPanel.vue` 调用 `worldbossApi`（全小写），但 `server.js` 注册的是 `worldBoss`（驼峰），且存在两个文件 `worldBoss.js` 和 `worldboss.js`
- **影响**: API路径可能是 `/api/worldBoss`（驼峰），但前端请求 `/api/worldboss`（全小写），导致404
- **证据**:
  ```
  server.js: app.use('/api/worldBoss', require('./routes/worldBoss'))  # 驼峰注册
  client/src/api/index.js: api.get('/worldboss')  # 全小写请求
  $ ls server/backend/routes/worldBoss.js  # EXISTS
  $ ls server/backend/routes/worldboss.js  # EXISTS
  ```
- **解决方案**:
  1. 统一路由命名：推荐全小写 `/api/worldboss`
  2. 确认 `worldboss.js` 文件内容（已存在）并使用它
  3. 更新前端 API 调用路径
- **工作量**: 小
- **负责人**: [B]

#### 9. 【前端】交易系统 TradePanel.vue 完全无API ⚠️
- **问题**: `TradePanel.vue` 有完整UI（市场/拍卖/我的上架），但只有 `console.log`，无任何真实API调用
- **影响**: 玩家无法在市场购买物品、无法参与拍卖、无法上架自己的物品
- **证据**: `TradePanel.vue` 的 `confirmBuy()`/`openBidDialog()`/`cancelListing()` 只有 `console.log`
- **解决方案**:
  1. 对接市场API（`/api/shop` 或新建 `/api/market`）
  2. 对接拍卖行API（`/api/auction`）
  3. 实现购买/出价/上架的完整逻辑
- **工作量**: 中
- **负责人**: [F]

#### 10. 【前后端】渡劫相性表缺失导致玩家决策困难 ⚠️
- **问题**: `TribulationPanel.vue` 调用了 `/api/tribulation/types` 和 `/api/tribulation/attempt`，但后端路由不匹配（`/api/tribulation` vs `/api/api/tribulation` 双前缀问题）
- **影响**: 玩家不知道哪种天劫最适合自己的灵根，渡劫失败率高导致挫败感
- **证据**:
  ```
  client/src/api/index.js: tribulationApi.preview = (params) => api.get('/api/tribulation/preview', { params })
  # → 请求路径: /api/api/tribulation/preview  ❌ 双前缀
  ```
- **解决方案**:
  1. 修复API路径（去掉前缀 `/api`，因为 `axios.create({ baseURL: API_BASE })` 已包含）
  2. 添加灵根相性推荐算法（金灵根→金劫成功率+15%等）
  3. 在渡劫面板显示相性表和推荐结果
- **工作量**: 小
- **负责人**: [B] + [F]

### P2 - 体验优化

#### 11. 【后端】灵石掉落数值过低，玩家初期极度匮乏
- **问题**: `balance_config.js` 战斗掉落 `base_min=5, base_max=20`，而一件普通装备强化需要50+灵石
- **影响**: 新玩家前10分钟体验极差，无法进行核心强化循环，快速流失
- **证据**: 基础掉落5-20灵石，但装备强化起步消耗50灵石，比例失衡
- **解决方案**:
  1. 新账号初始赠送: 灵石+1000，灵气+500（仅首次）
  2. 战斗掉落提升3倍: `base_min=15, base_max=60`
  3. 副本通关额外奖励: +500灵石（首次通关）
  4. 每日第一次分享: +100灵石
- **工作量**: 小
- **负责人**: [B]

## 五、新增方案摘要

### 方案A：宗门面板重构 [P0-5]
**背景**: SectPanel.vue 有UI但零API调用，GuildPanel.vue 有API但不是游戏主入口。  
**方案**: 以 SectPanel.vue 为游戏内宗门主入口，对接 `sectApi`，实现：
1. 宗门信息展示（名称/等级/成员数/资金）
2. 加入/创建宗门
3. 宗门捐献（灵石/经验）
4. 宗门建筑升级（藏经阁/炼器室/灵兽园）
5. 宗门技能学习
6. 宗门副本入口

### 方案B：宠物装备系统对接 [P0-7]
**背景**: PetEquipmentPanel.vue 有完整564行UI但无API调用，beast_equipment.js 后端存在。  
**方案**: 对接宠物装备API，实现：
1. 灵兽装备穿戴/卸下
2. 装备强化（攻击/生命/防御加成）
3. 装备套装激活效果
4. 装备品质标签（凡品/良品/精品/极品/神话）

### 方案C：灵石经济修复 [P1-4]
**背景**: 当前灵石掉落5-20/场，装备强化50+，比例1:3，严重供不应求。  
**方案**: 三管齐下：
1. **供给侧**: 战斗掉落提升3倍 → 15-60/场
2. **初始赠送**: 新账号首次登录 → +1000灵石
3. **目标激励**: 每日分享/签到/成就 → 额外灵石奖励

## 六、结论

本轮复验发现 **上轮4个P0问题零修复**，同时新增 **3个新P0阻断**（宗门/时装/宠物装备Stub）。根本原因是：
1. 大量前端面板是"UI壳"——有漂亮界面但无数据连接
2. 后端路由已注册但文件缺失（AbyssDungeon）
3. 付费入口（周卡/月卡/时装/VIP）全面Stub，直接影响收入

建议执行策略：
- **第一优先级**: 修复4个上轮P0（尤其是深渊API）
- **第二优先级**: 重构3个社交/付费Stub（宗门+时装+宠物装备）
- **第三优先级**: 修复API路径不一致（worldboss/ tribulation）
