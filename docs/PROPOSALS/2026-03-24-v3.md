## 二、本轮新发现问题清单

### P0 - 新阻断级问题

#### 13. 【后端+前端】sect.js 严重断裂 — 20+ API方法全部404 🚨

- **问题**: `ClanPanel.vue` (748行) 调用 `sectApi`，`sectApi` 定义了 20+ 个方法，但 `sect.js` 只实现了 3 个路由（`GET /`, `POST /create`, `POST /building/upgrade`）
- **影响**: 宗门捐献、成员管理、宗门技能、红包、管理员功能 **全部不可用**，核心社交系统实质性缺失
- **证据**:
  ```
  # sectApi 定义的方法（api/index.js）:
  getInfo, getBonus, getMembers, kickMember, promoteMember, transferLeader,
  getSkills, learnSkill, getDungeon, challengeDungeon, getRedPackets,
  sendRedPacket, claimRedPacket, getAdmin, donate → 全部 404

  # sect.js 实际路由（仅3个）:
  GET /          → 有
  POST /create   → 有
  POST /building/upgrade → 有
  其余 /info, /members, /bonus, /donate, /skills, /skill/learn, 
       /member/kick, /member/promote, /member/transfer, 
       /dungeon, /dungeon/challenge, /redpackets, /redpacket/*, /admin → 🚨 全部不存在

  # 实测:
  curl http://localhost:3001/api/sect/members?player_id=1
  → 404 Cannot GET /api/sect/members
  ```
- **现状**: `guild.js` 有完整的 guild 系统（create/join/leave/notice/skill/upgrade），但 `ClanPanel.vue` 调用的是 `sect.js` 而非 `guild.js`
- **解决方案**:
  1. **推荐**: 将 `ClanPanel.vue` 改为调用 `guildApi`（`guild.js` 已有完整后端）
  2. **备选**: 为 `sect.js` 补全所有 20+ 路由
- **工作量**: 中（推荐方案只需前端修改 API 来源）
- **负责人**: [F]

#### 14. 【后端】abyssDungeon.js 文件失踪 — 运行时炸弹 🚨

- **问题**: `server.js` 注册了 `app.use('/api/abyssDungeon', require('./routes/abyssDungeon'))`，API 也正常返回数据，但 `routes/abyssDungeon.js` 文件在磁盘上**不存在**
- **影响**: 这是典型的"运行时炸弹"——API 正常只是因为服务器进程还在使用内存中的旧代码。一旦重启，abyssDungeon 将完全 404。前端 `AbyssDungeonPanel.vue` (1150行) 依赖此API
- **证据**:
  ```
  $ ls /root/.openclaw/workspace/xundao-client/server/backend/routes/abyssDungeon*
  → (无输出，文件不存在)

  $ curl http://localhost:3001/api/abyssDungeon/config
  → {"success":true,"dungeons":[...]}  ← API 仍然工作！

  server.js:76 → app.use('/api/abyssDungeon', require('./routes/abyssDungeon'))
  ```
- **分析**: 服务器启动时加载了文件，之后文件被删除但进程未重启。abyssDungeon 功能实际上是 **内存残留**
- **解决方案**:
  1. 从 `abyss_api.js` 或 `daily_dungeon.js` 重建 `abyssDungeon.js`
  2. 或者将 abyssDungeon 合并到现有的 `daily_dungeon.js` 路由
  3. **立即执行**: 重启服务器前必须先修复
- **工作量**: 中
- **负责人**: [B]

#### 15. 【后端】tribulation 路由完全缺失 — 飞升系统断链 🚨

- **问题**: `TribulationPanel.vue` 调用 `fetch('/api/tribulation/types')`，`api/index.js` 定义了 `tribulationApi`，但 `server.js` 中**完全没有注册 tribulation 路由**，`routes/tribulation.js` 也不存在
- **影响**: 渡劫/飞升是修仙游戏核心玩法，玩家无法体验天劫选择、相性预览、渡劫尝试
- **证据**:
  ```
  server.js → 无 tribulation 注册
  $ ls routes/tribulation* → 不存在
  $ curl http://localhost:3001/api/tribulation/types → 404
  $ curl http://localhost:3001/api/tribulation/preview → 404
  ```
- **解决方案**:
  1. 创建 `routes/tribulation.js`，实现：
     - `GET /types` - 天劫类型列表（雷/火/冰/心魔/天罚）
     - `GET /preview` - 相性预览（根据玩家灵根推荐）
     - `POST /attempt` - 执行渡劫
  2. 参考 `server/game/tribulation_system.js` 获取游戏逻辑
  3. 更新 `server.js` 注册: `app.use('/api/tribulation', require('./routes/tribulation'))`
- **工作量**: 中
- **负责人**: [B]

## 六、结论

本轮发现的核心问题是**系统断裂**：
1. **数据层完好** (`guild.js`, `beast_equipment.js`, `tribulation_system.js` 等)
2. **API 定义层完好** (`sectApi`, `abyssApi`, `tribulationApi` 等)
3. **UI 层大量堆积** (`ClanPanel.vue` 748行、`AbyssDungeonPanel.vue` 1150行、`GuildPanel.vue` 1017行)
4. **路由实现层严重缺失** (`sect.js` 仅3路由、`abyssDungeon.js` 失踪、`tribulation.js` 不存在)

这解释了为什么"前端在喊、后端不应"——不是不想接，是没接上。

建议执行策略：
- **第一优先级**: 紧急修复3个P0（ClanPanel API切换、abyssDungeon重建、tribulation实现）
- **第二优先级**: 解决月卡/周卡混乱（P1-5）
- **第三优先级**: 师徒+婚姻系统（真正的社交深度）
